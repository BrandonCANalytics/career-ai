# BrandonCantrell.ai Design System

## Product Identity

BrandonCantrell.ai is not a traditional portfolio website.

It is an AI-enabled analytics product experience that showcases:

* analytics engineering
* AI-assisted workflows
* semantic modeling
* data product thinking
* modern business intelligence systems
* conversational analytics UX

The application should feel like a real internal analytics platform or AI-native SaaS product.

The user experience should communicate:

* technical maturity
* systems thinking
* product-minded engineering
* calm confidence
* clarity over decoration

---

# Primary Design Goals

1. Make the experience memorable
2. Make the experience feel premium
3. Make technical complexity feel approachable
4. Optimize for recruiter comprehension
5. Prioritize usability and information clarity
6. Showcase engineering taste and product thinking
7. Avoid generic developer portfolio aesthetics

---

# Visual Identity

## Desired Aesthetic

The application should feel similar to:

* ChatGPT
* OpenAI Playground
* Linear
* Vercel Dashboard
* Perplexity
* Cursor
* Retool
* modern AI-native SaaS applications

The visual style should be:

* dark
* calm
* technical
* minimal
* dense but readable
* modern without trend-chasing

The site should feel like:
“an AI analytics platform”
NOT:
“a flashy frontend portfolio”

---

# Anti-Goals

Avoid:

* giant gradient blobs
* overdesigned landing pages
* floating glassmorphism cards
* loud color palettes
* neon cyberpunk aesthetics
* excessive animations
* cheesy AI imagery
* generic startup templates
* giant marketing sections
* floating skill bars
* bouncing animations
* random decorative icons
* overuse of shadows
* dribbble-style gimmicks

Do not prioritize “creative” layouts over usability.

---

# Layout Philosophy

## General Principles

Favor:

* structure
* alignment
* hierarchy
* spacing consistency
* readability
* modularity

The application should feel:

* deliberate
* systematic
* composable

Whitespace should be intentional and restrained.

Layouts should feel:

* information-dense
* organized
* breathable

Avoid:

* excessive empty space
* giant hero sections
* awkward asymmetry
* visual clutter

---

# Page Structure

## Preferred Layout Patterns

Use:

* split-pane layouts
* command palette patterns
* conversational interfaces
* dashboard-style sections
* tabbed exploration
* expandable cards
* contextual side panels
* structured content blocks

Avoid:

* one-page scrolling portfolio designs
* giant marketing homepage structures
* excessive parallax
* landing-page-heavy layouts

---

# Typography

Typography should carry the visual sophistication.

## Typography Principles

Use:

* clean sans-serif fonts
* strong hierarchy
* restrained font weights
* consistent spacing
* readable line lengths

Typography should feel:

* modern
* calm
* intelligent
* technical

Prefer:

* medium font weights
* subtle hierarchy shifts
* compact spacing

Avoid:

* overly large headers
* decorative typography
* excessive italics
* condensed fonts
* loud typography systems

---

# Color System

## Base Palette

Background:

* near-black charcoal
* muted dark gray

Surface Colors:

* subtle layered grays
* low-contrast elevation

Primary Accent:

* soft blue
  OR
* muted emerald

Text:

* high-contrast whites
* muted grays for secondary content

Borders:

* subtle low-contrast separators

Avoid:

* pure black backgrounds
* high saturation
* rainbow accents
* colorful gradients
* bright purple/pink startup palettes

The UI should feel calm and mature.

---

# Component Philosophy

## Preferred Components

Use:

* command palettes
* chat interfaces
* prompt suggestion chips
* expandable technical cards
* tabbed views
* searchable lists
* architecture diagrams
* markdown rendering
* code snippets
* metric callouts
* timeline systems
* semantic relationship views

Components should feel:

* modular
* reusable
* composable
* product-oriented

---

# Cards

Cards should:

* have restrained elevation
* subtle borders
* moderate rounded corners
* strong internal spacing

Avoid:

* floating card overload
* giant shadows
* random card rotations
* glassmorphism

---

# Buttons

Buttons should feel:

* understated
* functional
* modern

Use:

* subtle hover states
* restrained colors
* minimal gradients

Avoid:

* oversized CTAs
* flashy effects
* aggressive hover animations

---

# Motion & Animation

Motion should:

* reinforce usability
* feel fast
* feel intentional

Preferred Motion:

* subtle fades
* smooth streaming
* lightweight hover transitions
* contextual transitions
* progressive disclosure

Avoid:

* bounce effects
* dramatic entrance animations
* distracting motion
* slow transitions
* gimmicky page animations

Animation should support clarity, not spectacle.

---

# AI Interface Principles

The conversational interface is the centerpiece of the application.

It should feel:

* intelligent
* responsive
* grounded
* recruiter-friendly

The interface should resemble:

* ChatGPT
* Perplexity
* Open WebUI
* Cursor chat

NOT:

* a toy chatbot
* customer support software

---

# Suggested Prompt UX

Suggested prompts should:

* feel strategic
* feel curated
* feel recruiter-oriented

Examples:

* “How does Brandon approach analytics engineering?”
* “What AI-enabled systems has Brandon built?”
* “Explain Brandon’s semantic modeling experience”
* “Show examples of stakeholder-facing data products”

Prompt chips should:

* feel clickable
* feel polished
* feel lightweight

Avoid:

* excessive emoji usage
* gimmicky prompt wording

---

# Content Philosophy

The site should communicate:

* impact
* systems thinking
* product maturity
* technical breadth
* stakeholder empathy

Avoid generic resume language.

Favor:

* architecture decisions
* technical tradeoffs
* measurable impact
* product outcomes
* operational improvements

---

# Recruiter Experience

Optimize for:

* fast comprehension
* memorable interaction
* trust
* clarity

The user should quickly understand:

* what Brandon builds
* how Brandon thinks
* where Brandon fits technically
* why Brandon is differentiated

The experience should reduce friction and encourage exploration.

---

# Technical Tone

The application should feel:

* modern
* pragmatic
* engineered
* AI-native

Avoid:

* hackathon vibes
* crypto aesthetics
* overengineered startup energy

The application should communicate:
“I build real analytics products.”

---

# Personalization Philosophy

Role-based personalization should feel:

* subtle
* intelligent
* contextual

It should never feel:

* gimmicky
* creepy
* over-personalized

Changing role personas should:

* reprioritize content
* alter suggested prompts
* shift featured projects
* adjust narrative emphasis

without dramatically changing the overall visual system.

---

# Engineering Principles

Favor:

* clean architecture
* composable components
* maintainability
* readability
* deterministic logic

Avoid:

* unnecessary abstractions
* premature optimization
* agentic complexity
* overuse of libraries

The application should feel intentionally engineered.

---

# Final Design Goal

The final experience should make recruiters feel:

“This person understands modern AI-enabled analytics systems and product engineering better than most data candidates.”

The application itself should function as a proof-of-work artifact.
